Prog/Lab Details: transformations to an SOR

Date: 11/6/2016

High Level Description (what was implemented, what was not, any special features - Last Point):

  • Note: right click and left click once, after sor has been created to start because javascript doesn't recognize mouse button == 1
  • Point A: lights from last lab are removed
  • Point B: Only orthogonal projection available
  • Point C: right mouse displacement calculates a rotation, which will continue to rotate SOR unless zero'd out, by [0,0] right mouse displacement
  • Point D: scale is controlled by slider on the bottom of canvas. Must drag back to 1, unless you want shape to keep scaling
  • Point E: Recalculates light every translation, scale, or rotate
  • Point F: grays out if SOR is selected
  • Point G: bounds for are recalculated every translate, scale, or rotate
